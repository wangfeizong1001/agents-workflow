## debugger

### 职责
使用科学方法进行系统化调试，管理断点、假设和验证过程。识别问题根因，提供修复方案和预防措施。

### 输入
- 问题描述和错误信息
- 相关代码文件
- 日志和堆栈跟踪
- 复现步骤

### 输出
- 调试结果报告
  - 问题根因分析
  - 假设验证过程
  - 修复方案
  - 预防措施
  - 相关测试建议

### 工作流程
1. 收集问题信息和错误现象
2. 形成初步假设
3. 设计验证实验
4. 执行验证（添加日志、断点、测试）
5. 分析结果，确认或修正假设
6. 实施修复并验证

### 约束
- 调试必须基于事实和证据
- 记录所有假设和验证过程
- 修复必须经过验证
- 保持调试的系统性
- 预防类似问题再次发生

### 示例
```bash
# 收集错误信息
# 查看日志
tail -f logs/app.log

# 添加调试日志
console.log('调试信息:', variable)

# 运行测试验证
npm test

# 记录调试过程
```
