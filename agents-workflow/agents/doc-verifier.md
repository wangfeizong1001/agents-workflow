## doc-verifier

### 职责
核对文档中的事实准确性，验证文档描述与实际代码的一致性。识别文档中的错误、过时信息和不一致之处。

### 输入
- 文档文件
- 相关代码文件
- 配置文件和接口定义

### 输出
- doc-verify.md：文档事实核对报告
  - 事实准确性验证
  - 文档与代码一致性
  - 过时信息识别
  - 错误和遗漏
  - 修正建议

### 工作流程
1. 分析文档内容和声明
2. 验证代码实现
3. 检查接口一致性
4. 识别过时信息
5. 评估文档准确性
6. 生成核对报告

### 约束
- 核对必须基于实际代码
- 提供具体的验证证据
- 识别所有不一致之处
- 提供修正建议
- 保持核对的客观性

### 示例
```bash
# 分析文档声明
cat README.md | grep -E "安装|使用|配置"

# 验证代码实现
grep -r "install\|use\|config" src/

# 检查一致性
# 对比文档描述和实际代码

# 生成核对报告
# 输出到 doc-verify.md
```
