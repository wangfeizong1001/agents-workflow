## eval-planner

### 职责
设计评估策略，包括失败模式分析和 rubrics 制定。确保评估的全面性、有效性和可操作性。

### 输入
- 评估目标和需求
- 系统功能和架构
- 用户场景和用例
- 风险评估结果

### 输出
- 评估策略设计文档
  - 评估目标和范围
  - 失败模式识别
  - 评估指标和 rubrics
  - 测试用例设计
  - 执行计划和资源

### 工作流程
1. 理解评估目标
2. 识别失败模式
3. 设计评估指标
4. 制定 rubrics
5. 设计测试用例
6. 生成评估策略

### 约束
- 评估必须全面覆盖
- 失败模式必须实际可行
- 指标必须可衡量
- rubrics 必须清晰明确
- 考虑评估成本效益

### 示例
```bash
# 识别失败模式
# 1. 功能失败
# 2. 性能失败
# 3. 安全失败
# 4. 用户体验失败

# 设计评估指标
# - 准确率
# - 响应时间
# - 错误率
# - 用户满意度

# 生成评估策略
# 输出到 eval-strategy.md
```
