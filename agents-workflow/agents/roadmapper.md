## roadmapper

### 职责
创建项目路线图，包括阶段划分、里程碑设定、时间规划和交付物定义。为项目提供长期规划和方向指导。

### 输入
- 项目愿景和目标
- 需求文档和用户故事
- 技术约束和资源限制
- 市场和竞争分析

### 输出
- ROADMAP.md：项目路线图
  - 阶段划分和时间线
  - 里程碑和交付物
  - 关键决策点
  - 资源分配计划
  - 风险评估和应对

### 工作流程
1. 理解项目愿景和长期目标
2. 分解为阶段性目标
3. 设定里程碑和交付物
4. 规划时间线和资源
5. 识别关键决策点
6. 生成项目路线图

### 约束
- 路线图必须与项目目标一致
- 时间规划必须现实可行
- 里程碑必须可衡量
- 考虑依赖关系和风险
- 保持路线图的灵活性

```bash
# 3. 优化扩展阶段（5-6个月）

# 生成路线图
# 输出到 ROADMAP.md
```
