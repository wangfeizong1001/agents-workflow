## /spike 命令

### 用法
/spike [技术问题]

### 参数
- `--timebox`: 时间限制
- `--approach`: 探索方法
- `--deliverable`: 交付物

### 行为
1. 定义探索问题
2. 设置时间限制
3. 执行技术探索
4. 记录发现
5. 生成探索结论

### 示例
```
/spike "GraphQL vs REST" --timebox 2hours --deliverable comparison
```

### 输出
技术探索报告和决策建议
