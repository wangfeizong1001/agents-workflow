## brainstorming

### 用途
在任何创造性工作之前必须使用此技能。通过 Socratic 追问式方法探索用户意图、需求和设计。在实现之前先理解"为什么"和"是什么"。

### 使用方式
1. 听取用户初步想法
2. 追问核心问题（为什么做？给谁做？成功是什么样？）
3. 探索替代方案
4. 澄清边界条件
5. 生成设计文档

### 输入
- 初始想法或需求描述
- 可选：相关背景信息

### 输出
- 设计文档
- 明确的需求清单
- 待解决问题列表
- 下一步行动计划

### 示例
```
用户: 我想添加一个搜索功能

追问:
1. 搜索什么内容？文章？用户？产品？
2. 谁在搜索？所有用户还是特定角色？
3. 搜索延迟要求是多少？实时还是可以接受几秒？
4. 需要模糊匹配还是精确匹配？
5. 有现有的搜索引擎基础架构吗？

输出: 搜索功能设计文档 v1.0
```
