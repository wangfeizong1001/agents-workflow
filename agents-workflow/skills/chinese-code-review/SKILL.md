## chinese-code-review

### 用途
中文 review 沟通参考——话术模板、分级标注（必须修复/建议修改/仅供参考）、国内团队常见反模式应对。

### 使用方式
1. 选择审查级别
2. 使用标准话术模板
3. 标注问题等级
4. 提供修复建议
5. 沟通最佳实践

### 输入
- 代码变更
- 审查焦点
- 团队规范

### 输出
- 审查报告
- 分级标注
- 修复建议

### 示例
```markdown
## 代码审查

### 🔴 必须修复
1. SQL 注入风险（第 45 行）
   - 建议: 使用参数化查询

### 🟡 建议修改
2. 函数过长（第 100-200 行）
   - 建议: 拆分为更小的函数

### 🟢 仅供参考
3. 变量命名可以更清晰
   - 建议: `data` → `userData`

### 沟通话术
- "这个实现有个安全隐患..."
- "建议考虑..."
- "整体不错，小建议..."
```
