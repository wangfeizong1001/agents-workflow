## executing-plans

### 用途
当你有一份书面实现计划需要在单独的会话中执行，并设有审查检查点时使用。

### 使用方式
1. 读取实现计划
2. 识别检查点
3. 按步骤执行
4. 在检查点暂停审查
5. 继续或调整执行

### 输入
- 实现计划文档
- 可选：起始检查点
- 可选：执行约束

### 输出
- 执行进度
- 检查点报告
- 代码变更
- 验证结果

### 示例
```markdown
## 执行计划

### 检查点 1: 基础设施
- [x] 数据库 schema
- [x] 配置管理
- [x] 日志系统
→ 审查: 通过

### 检查点 2: 核心功能
- [ ] API 端点
- [ ] 业务逻辑
- [ ] 数据验证
→ 审查: 进行中

### 检查点 3: 测试
- [ ] 单元测试
- [ ] 集成测试
→ 待执行
```
