## gstack-openclaw-office-hours

### 用途
Use when asked to brainstorm, evaluate whether an idea is worth building, run office hours, or think through a new product idea or design direction before any code is written.

### 使用方式
1. 收集产品想法
2. 追问核心问题
3. 评估可行性
4. 探索替代方案
5. 生成设计文档

### 输入
- 产品想法
- 目标用户
- 约束条件

### 输出
- 评估报告
- 设计文档
- 下一步行动

### 示例
```markdown
## Office Hours - 新功能评估

### 想法
"添加 AI 辅助写作功能"

### 追问
1. 目标用户是谁？
2. 现有解决方案是什么？
3. 我们的优势是什么？

### 评估
- 需求强度: 8/10
- 技术可行性: 7/10
- 竞争激烈度: 9/10

### 结论
值得构建，但需要差异化定位
```
