## investigate

### 用途
系统化调试与根因分析。四个阶段：调查、分析、假设、实施。铁律：未进行根因调查之前，不做任何修复。

### 使用方式
1. **调查阶段**: 收集信息、复现问题
2. **分析阶段**: 分析日志、追踪代码路径
3. **假设阶段**: 提出可能的根因
4. **实施阶段**: 验证假设并修复

### 输入
- 问题描述
- 错误日志
- 复现步骤

### 输出
- 根因分析报告
- 修复方案
- 预防措施

### 示例
```markdown
## 根因分析 - 登录失败

### 调查
- 现象: 用户报告无法登录
- 复现: 使用正确密码仍返回 401

### 分析
- 日志: "Token validation failed"
- 代码路径: auth.ts:validateToken

### 假设
1. 时钟不同步导致 token 过期判断错误
2. 密钥配置错误

### 验证
- 检查服务器时间: 正常
- 检查密钥: 发现使用了错误的密钥

### 修复
- 更新 JWT_SECRET 环境变量
```
