## plan-eng-review

### 用途
工程经理模式的计划审查。锁定执行计划——架构、数据流、图表、边界条件、测试覆盖、性能。以互动方式逐项处理问题，给出有倾向性的建议。

### 使用方式
1. 读取计划文档
2. 评估架构设计
3. 分析数据流
4. 检查边界条件
5. 评估测试覆盖
6. 识别性能风险
7. 输出审查报告

### 输入
- 计划文档
- 架构图（可选）
- 现有代码上下文

### 输出
- 架构评估
- 数据流分析
- 边界条件清单
- 测试覆盖建议
- 性能风险

### 示例
```markdown
## 工程审查 - 用户认证系统

### 架构
- ✅ JWT 无状态设计
- ⚠️ 缺少 Token 黑名单机制

### 数据流
- 注册: 验证 → 哈希 → 存储 → 返回 Token
- 登录: 验证 → 比对 → 生成 Token → 返回

### 边界条件
- [ ] Token 过期处理
- [ ] 并发登录限制
- [ ] 密码强度验证

### 测试覆盖
- 单元测试: 85%
- 集成测试: 60%
- 建议: 增加边界测试
```
