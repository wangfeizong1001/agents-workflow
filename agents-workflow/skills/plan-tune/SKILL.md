## plan-tune

### 用途
skill 的自调问题敏感度 + 开发者心理画像。审查哪些 AskUserQuestion 提示在 skill 技能中触发，为每个问题设置偏好，检查双轨画像，并启用/禁用问题调优。

### 使用方式
1. 分析问题触发模式
2. 评估问题必要性
3. 设置问题偏好
4. 检查画像一致性
5. 调优问题频率

### 输入
- 技能文件
- 问题历史
- 用户反馈

### 输出
- 问题偏好配置
- 画像分析
- 调优建议

### 示例
```markdown
## 问题调优分析

### 技能: brainstorming

### 问题历史
| 问题 | 触发次数 | 用户满意度 |
|------|----------|------------|
| 目标用户是谁？ | 15 | 高 |
| 有竞品参考吗？ | 8 | 中 |
| 预算范围？ | 3 | 低 |

### 调优建议
- "目标用户是谁？" → always-ask
- "有竞品参考吗？" → ask-only-for-one-way
- "预算范围？" → never-ask

### 画像
- 偏好: 直接给出方案
- 痛点: 过多澄清问题
- 风格: 快速迭代
```
