## prd

### 用途
生成产品需求文档（PRD）。用于规划功能、启动新项目或被要求创建 PRD 时。包含背景、目标、用户故事、功能需求、非功能需求、成功指标等。

### 使用方式
1. 收集产品背景信息
2. 明确目标和成功指标
3. 编写用户故事
4. 定义功能需求
5. 确定非功能需求
6. 输出完整 PRD

### 输入
- 产品/功能描述
- 目标用户群体
- 业务目标
- 可选：竞品分析

### 输出
- 完整的 PRD 文档
- 用户故事列表
- 功能优先级矩阵
- 成功指标定义

### 示例
```markdown
# PRD: 用户认证系统

## 背景
当前系统缺少用户认证，需要实现安全的身份验证机制

## 目标
- 支持 10,000+ 并发用户认证
- 登录成功率 > 99.9%
- 平均响应时间 < 200ms

## 用户故事
1. 作为新用户，我可以注册账号
2. 作为注册用户，我可以登录系统
3. 作为登录用户，我可以退出登录

## 功能需求
- FR-1: 邮箱/密码注册
- FR-2: 邮箱/密码登录
- FR-3: JWT Token 管理
```
