## writing-plans

### 用途
当你有规格说明或需求用于多步骤任务时使用，在动手写代码之前。将需求分解为可执行的步骤，定义检查点和验证标准。

### 使用方式
1. 分析规格说明
2. 识别依赖关系
3. 分解为原子步骤
4. 定义检查点
5. 编写验证标准
6. 输出执行计划

### 输入
- 功能规格说明
- 需求文档
- 可选：现有代码上下文

### 输出
- 分步执行计划
- 依赖关系图
- 检查点定义
- 验证标准

### 示例
```markdown
## 执行计划: 用户认证系统

### Phase 1: 基础设施 (Day 1)
- [ ] 创建数据库 schema
- [ ] 设置 JWT 密钥管理
- [ ] 验证: 数据库迁移成功

### Phase 2: 核心功能 (Day 2-3)
- [ ] 实现注册端点
- [ ] 实现登录端点
- [ ] 实现 Token 刷新
- [ ] 验证: 单元测试通过

### Phase 3: 集成 (Day 4)
- [ ] 添加认证中间件
- [ ] 集成到现有路由
- [ ] 验证: API 测试通过
```
