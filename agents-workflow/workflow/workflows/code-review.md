## 代码审查

> **触发条件**：需要审查代码质量时使用
> **预计耗时**：20分钟
> **产出物**：代码审查报告

### 阶段 1：准备（5分钟）

- ① git-diff → 查看代码变更
- ② review → 准备审查材料
- 门禁：变更范围已明确

### 阶段 2：审查（10分钟）

- ① code-review → 执行代码审查
- ② chinese-code-review → 中文审查沟通
- 门禁：审查意见已记录

### 阶段 3：反馈（5分钟）

- ① session-report → 生成审查报告
- ② learn → 记录审查经验
- 门禁：审查报告已保存

### 回滚方案
- 根据审查意见修复问题
