## 编辑阶段

> **触发条件**：需要修改阶段计划或实现时使用
> **预计耗时**：15分钟
> **产出物**：修改后的阶段内容

### 阶段 1：分析（5分钟）

- ① explore → 分析需要修改的内容
- ② investigate → 调查问题原因
- 门禁：修改范围已确定

### 阶段 2：修改（7分钟）

- ① edit-phase → 执行修改操作
- ② apply-change → 应用变更
- 门禁：修改已完成

### 阶段 3：验证（3分钟）

- ① verify-change → 验证修改正确
- ② health → 检查影响范围
- 门禁：修改验证通过

### 回滚方案
- 使用 git 撤销修改
