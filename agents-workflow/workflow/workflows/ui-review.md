## UI 审查

> **触发条件**：需要审查 UI 设计时使用
> **预计耗时**：15分钟
> **产出物**：UI 审查报告

### 阶段 1：截图（5分钟）

- ① browse → 截取界面截图
- ② design-review → 准备审查
- 门禁：截图已获取

### 阶段 2：审查（7分钟）

- ① ui-review → 执行 UI 审查
- ② plan-design-review → 设计评审
- 门禁：审查意见已记录

### 阶段 3：反馈（3分钟）

- ① session-report → 生成审查报告
- ② apply-change → 应用修复
- 门禁：报告已保存

### 回滚方案
- 根据审查意见修改 UI
